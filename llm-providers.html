LLM Providers

Use Groq or OpenAI out of the box, or plug in any LLM in roughly 10 lines of code.

Built-in Providers

ProviderString FormatExampleAPI Key Env Var
Groq Groq:model-name Groq:qwen/qwen3-32b GROQ_API_KEY
OpenAI OpenAI:model-name OpenAI:gpt-4o OPENAI_API_KEY
Custom MyVendor:model-name MyVendor:my-model Your own
🆓
Groq has a free tier Sign up at console.groq.com — no credit card required to start. Models like qwen/qwen3-32b are fast and capable for hyperparameter space generation.
🔓
Model name validation is not enforced Any model name string is accepted. The provider API returns a clear error if the model doesn't exist — this allows using newly released models without a library update.

Register a Custom LLM Provider

Implement BaseLLMHandler and override _call_llm(). The base class handles retry logic, JSON repair, schema validation, and fallback — you only need to implement the API call itself.

from mltunex.ai_handler.llm_handler_base import (
    BaseLLMHandler, LLMHandlerConfig, LLMHandlerRegistry
)

class MyVendorHandler(BaseLLMHandler):
    provider_name = "MyVendor"   # must match the prefix in Provider:ModelName

    def _call_llm(self, prompt_vars: dict) -> str:
        # Call your own API. Return raw text.
        # The base class handles: retry, JSON repair, schema validation, fallback.
        return my_api.complete(
            system = prompt_vars["HyperparameterResponsePrompt"],
            user   = prompt_vars["Data_Profile"],
        )

LLMHandlerRegistry.register(MyVendorHandler)

# Use it:
config = OrchestratorConfig(
    ...,
    model_provider_model_name = "MyVendor:my-model",
)

Capturing Token Usage

To have token usage appear in tuning_report.md, override _call_llm_with_obj() instead, returning a tuple of (response_text, response_object). The base class then calls token_usage.record(response_obj).

class MyVendorHandler(BaseLLMHandler):
    provider_name = "MyVendor"

    def _call_llm_with_obj(self, prompt_vars: dict):
        resp = my_api.complete(
            system = prompt_vars["HyperparameterResponsePrompt"],
            user   = prompt_vars["Data_Profile"],
        )
        return resp.text, resp   # base class calls token_usage.record(resp)

How the Provider String is Parsed

The model_provider_model_name string is split on the first colon. The left side (Provider) is looked up in LLMHandlerRegistry. The right side (model-name) is passed to the handler's LLMHandlerConfig.

# "Groq:qwen/qwen3-32b"
# → provider = "Groq"    → looks up GroqHandler in registry
# → model    = "qwen/qwen3-32b"  → set on LLMHandlerConfig.model_name
