Troubleshooting

Common errors and exactly how to fix them.

UnicodeEncodeError on Windows terminal

Fix: Set terminal to UTF-8

Windows terminals default to a code page that can't represent Unicode characters used in MLTuneX output.

chcp 65001        # set console to UTF-8
# or
set PYTHONIOENCODING=utf-8
--llm is required when AI tuning is enabled

Fix: Provide --llm or use --no-tune

When tune_models=True (the default), an LLM provider is required. Either specify one or skip tuning:

# Option A: provide the LLM
mltunex --data data.csv --target label --task classification \
        --llm "Groq:qwen/qwen3-32b"

# Option B: skip AI tuning (no API key needed)
mltunex --data data.csv --target label --task classification \
        --no-tune
LightGBM [Warning] or [Info] messages in terminal

Fix: Set env vars before importing MLTuneX

MLTuneX sets LIGHTGBM_VERBOSITY=-1 at import time. In Jupyter, the kernel may pre-import LightGBM before MLTuneX runs. Set the env vars first:

import os
os.environ["LIGHTGBM_VERBOSITY"] = "-1"
os.environ["LIGHTGBM_SILENT"]    = "1"

from mltunex.orchestrator import MLTuneXOrchestrator, OrchestratorConfig

Or from the terminal before launching Jupyter:

set LIGHTGBM_VERBOSITY=-1      # Windows
export LIGHTGBM_VERBOSITY=-1   # Linux / macOS
LLM returns invalid JSON — fallback fires

No action required — but here's what's happening

After 3 failed validation attempts, MLTuneX automatically uses predefined search spaces for 28 known models. The run continues. Check pipeline.log in your experiment directory for the exact error from each attempt.

See the AI Validation page for a full explanation of the three-layer system.

Exception ignored in sys.unraisablehook from rich.console in Jupyter

Fix: Update to the latest version of MLTuneX

This was caused by Rich's Live display conflicting with Jupyter's output system. MLTuneX now auto-detects ZMQInteractiveShell and uses plain text output instead. Update your installation:

pip install --upgrade MLTuneX
pyarrow.lib.ArrowTypeError in Streamlit

Fix: Cast mixed-type columns to string

This is handled internally for hyperparameter display. If you see it from your own data, cast the problematic column:

df["col"] = df["col"].astype(str)
pipeline.log not created

Fix: Set terminal encoding on Windows

On Windows, ensure the terminal is set to UTF-8 (chcp 65001) before launching. The pipeline logger avoids non-ASCII characters in log messages, but the terminal encoding must still support file creation.

ModuleNotFoundError: No module named 'lightgbm'

Fix: Install LightGBM

pip install lightgbm
Out of memory with parallel training

Fix: Reduce worker count or use sequential mode

Parallel training spawns one process per model simultaneously. Reduce the worker count or fall back to the default sequential mode:

# Reduced workers
config = OrchestratorConfig(..., parallel_training=True, n_jobs=2)

# Sequential (default)
config = OrchestratorConfig(..., parallel_training=False)
