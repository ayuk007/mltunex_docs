Usage

Python API

All the ways to drive MLTuneX from Python — from minimal one-liners to fully customised runs.

Minimal Classification

from mltunex.orchestrator import MLTuneXOrchestrator, OrchestratorConfig

config = OrchestratorConfig(
    source                    = "data/titanic.csv",
    target_column             = "Survived",
    task_type                 = "classification",
    model_provider_model_name = "Groq:qwen/qwen3-32b",
)
MLTuneXOrchestrator(config).run()

Regression with Full Options

config = OrchestratorConfig(
    source          = "data/housing.parquet",
    target_column   = "SalePrice",
    task_type       = "regression",
    test_size       = 0.15,

    model_provider_model_name = "OpenAI:gpt-4o",

    preprocess        = True,
    parallel_training = True,
    n_jobs            = 8,

    tune_models  = True,
    n_trials     = 50,

    selection_strategy        = "stability",
    top_k                     = 5,
    selector_primary_metric   = "R2",
    selector_stability_weight = 0.3,

    log_dir         = "logs/",
    experiment_name = "housing_v1",
)
MLTuneXOrchestrator(config).run()

Skip Tuning — No API Key Required

Set tune_models=False to select and save the top-K models without calling any LLM.

config = OrchestratorConfig(
    source        = "data/fraud.csv",
    target_column = "is_fraud",
    task_type     = "classification",

    tune_models        = False,
    selection_strategy = "generalization",
    selector_train_metric = "Train_Accuracy",
    top_k              = 3,
)
MLTuneXOrchestrator(config).run()

Pass an In-memory DataFrame

You can hand MLTuneX a pandas.DataFrame directly instead of a file path.

import pandas as pd

df = pd.read_csv("data.csv")
config = OrchestratorConfig(
    source        = df,           # DataFrame passed directly
    target_column = "label",
    task_type     = "classification",
    model_provider_model_name = "Groq:qwen/qwen3-32b",
)
MLTuneXOrchestrator(config).run()

Jupyter / Colab

No extra configuration is needed. MLTuneX auto-detects the Jupyter environment (ZMQInteractiveShell) and switches the training progress table to plain text output, avoiding Rich Live display conflicts.

from mltunex.orchestrator import MLTuneXOrchestrator, OrchestratorConfig

# With AI tuning
config = OrchestratorConfig(
    source        = "titanic.csv",
    target_column = "Survived",
    task_type     = "classification",
    model_provider_model_name = "Groq:qwen/qwen3-32b",
)
MLTuneXOrchestrator(config).run()

# Without AI tuning (no API key needed)
config_notune = OrchestratorConfig(
    source        = "titanic.csv",
    target_column = "Survived",
    task_type     = "classification",
    tune_models   = False,
)
MLTuneXOrchestrator(config_notune).run()

Inference with Saved Artefacts

After a run, both the fitted preprocessing pipeline and the best model are saved as .joblib files. Load them to run predictions on new data:

import joblib, pandas as pd

# Load the saved pipeline and model
pipeline = joblib.load("logs/exp_20250502_143022/models/preprocessing_pipeline.joblib")
model    = joblib.load("logs/exp_20250502_143022/models/RandomForestClassifier.joblib")

# Predict on new data
X_new = pd.read_csv("new_data.csv").drop(columns=["target"])
preds = model.predict(pipeline.transform(X_new))
💡
Always apply the pipeline first The preprocessing pipeline must be applied to new data before prediction. It encodes and scales the data exactly as during training.

Selection Strategies

MLTuneX ships with three model selection strategies. Pass one of the following strings to selection_strategy:

StrategyDescriptionKey Config
topk default Selects the K models with the highest primary metric score. selector_primary_metric
stability Penalises models with high variance across cross-validation folds. selector_stability_weight
generalization Penalises models with a large train/test performance gap. selector_train_metric, selector_gap_penalty
