Core Concepts

Configuration Reference

Every field available in OrchestratorConfig, SelectorConfig, and LLMHandlerConfig.

OrchestratorConfig

The primary configuration object. Pass it to MLTuneXOrchestrator(config).run().

Required Fields

FieldTypeDescription
sourcestr | pd.DataFrameDataset file path or an in-memory DataFrame.
target_columnstrName of the target column to predict.
task_typestr"classification" or "regression".
model_provider_model_namestrFormat: "Provider:ModelName". Defaults to "Groq:none". Only required when tune_models=True.

I/O and Logging

FieldDefaultDescription
log_dir"logs/"Root directory where experiment folders are created.
experiment_name"exp"Tag prepended to the timestamped experiment folder name.
result_csv_path"results/"Directory for the evaluation CSV output.
model_dir_path"models/"Fallback only. At runtime, models are always saved inside <experiment_dir>/models/.

Training

FieldDefaultDescription
test_size0.2Fraction of data held out for evaluation.
preprocessTrueBuild and apply the adaptive preprocessing pipeline.
parallel_trainingFalseTrain models in a multiprocessing pool simultaneously.
n_jobs-1Worker count. -1 uses all available CPUs.
models_library"sklearn"Model library backend.

Tuning

FieldDefaultDescription
tune_modelsTrueRun AI-guided Optuna tuning. Set False to skip LLM and save top-K models directly.
n_trials25Number of Optuna trials per candidate model.
optimizer_method"optuna"Optimizer backend. Extendable via OptimizerFactory.register().
hyperparameter_framework"Optuna"Framework name looked up in ResponseSchemaRegistry to format AI prompt instructions.

Model Selection

FieldDefaultDescription
profiling_strategy"extended""basic" or "extended" data profiling depth.
selection_strategy"topk""topk", "stability", or "generalization".
top_k3Number of candidates forwarded to the optimizer.
selector_primary_metricNoneOverride primary metric. Auto-derives: Accuracy for classification, R2 for regression.
selector_stability_weight0.2Instability penalty weight for the stability strategy.
selector_train_metricNoneTrain-set metric column for the generalization strategy.
selector_gap_penalty0.5Train/test gap penalty weight for the generalization strategy.

SelectorConfig

All three selector strategies share the same constructor signature. The factory always receives a SelectorConfig so the orchestrator never needs to know which strategy is active.

from mltunex.model_registry.selector import SelectorConfig, ModelSelectorFactory

cfg = SelectorConfig(
    primary_metric   = "Accuracy",
    stability_weight = 0.3,              # used by StabilityAwareSelector
    train_metric     = "Train_Accuracy", # used by GeneralizationSelector
    gap_penalty      = 0.4,             # used by GeneralizationSelector
)
selector = ModelSelectorFactory.create("stability", config=cfg)
top_df   = selector.select(results_df, k=3)

LLMHandlerConfig

Provider-agnostic configuration used by all built-in and custom LLM handlers:

from mltunex.ai_handler.llm_handler_base import LLMHandlerConfig

config = LLMHandlerConfig(
    model_name    = "qwen/qwen3-32b",
    temperature   = 0.0,
    system_prompt = "...",    # must contain all 4 required placeholders
    framework     = "Optuna", # looked up in ResponseSchemaRegistry
)
💡
Model name validation is not enforced Any model name string is accepted. The LLM provider API returns a clear error if the model is unknown — this allows using newly released models without updating MLTuneX.
