Usage

Streamlit UI

A fully visual, no-code interface for MLTuneX — upload data, configure, and launch from your browser.

Launching the UI

mltunex ui
# or equivalently
python -m mltunex.ui

Opens at http://localhost:8501. The interface uses tabs instead of a sidebar, avoiding sidebar collapse or visibility issues.

Tab Overview

Tab 1

⚙️ Configure

Upload data, select target column from a dropdown, configure all pipeline options, paste API key securely, and launch the run.

Tab 2

▶️ Run

Live event log, model training progress bar, incremental results table, final metrics, and saved artefact paths.

Tab 3

🔍 Profile

Dataset overview metrics, missing data table with severity indicators, skewness bar chart, and target distribution chart.

Tab 4

📊 Results

Sortable model leaderboard plus a primary-metric bar chart for visual comparison.

Tab 5

🏆 Selection

Strategy card with description, plus the selected candidates table showing which models were forwarded to the optimiser.

Tab 6

🎯 Tuning

Best score metrics, optimal parameters table, score progression line chart, and full ranked trial history.

Tab 7

📄 Reports

Inline rendered Markdown reports, Excel preview with download, and pipeline log viewer.

UI Notes

  • The target column is a dropdown auto-populated from the uploaded file's columns — no typing required.
  • The AI Advisor section (provider, model, API key) is hidden automatically when the "AI hyperparameter tuning" toggle is off.
  • All print() output and C-level library noise (LightGBM, XGBoost, Optuna) is fully suppressed from the terminal — nothing leaks to the Streamlit server console.
  • API keys are kept in session memory only — never written to disk or logs.
🔐
API key security Keys entered in the UI exist only in the browser session. They are never persisted to any file or environment variable.
